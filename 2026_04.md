## Codex Computer Use

- UX: hidden pointer/app-control lets automation click without disturbing normal mouse usage.
- Alma: latest Alma includes Codex app Computer Use; usage reportedly exceeds Longxia + Claude Cowork.
- OSS adjacent: browser-harness self-healing Chrome CDP harness; agents patch/extend capabilities live.
- Open Computer Use: OSS clone exposes 9 tools: `list_apps`, `get_app_state`, `click`, `perform_secondary_action`, `scroll`, `drag`, `type_text`, `press_key`, `set_value`. Ref: https://github.com/iFurySt/open-codex-computer-use
- macOS stack: likely MCP tool surface + native helper. Uses Accessibility + Screen Recording, AX tree/window snapshot, ScreenCaptureKit PNG image block, in-process snapshot cache for `element_index`.
- macOS input model: prefer semantic AX actions (`AXPress`, `AXConfirm`, `AXOpen`, `AXUIElementSetAttributeValue`) before synthetic pointer. Coordinate click maps screenshot pixels to window points/Quartz coords; avoids moving real hardware cursor.
- Background click mechanism: synthesize `NSEvent`, extract `CGEvent`, set fields, post with `CGEvent.postToPid(pid)` to target process. Needs `CGWindowID` from `CGWindowListCopyWindowInfo`; set window-under-pointer fields 91/92; use private `CGEventSetWindowLocation` via `dlsym` for window-local point. Ref: https://github.com/Lakr233/bgclick-rev-skill/blob/main/bgclick-rev-skill.md
- Background app bypass: when target app inactive, set `event.flags = kCGEventFlagMaskCommand` (`0x00100000`) as WindowServer filter bypass. Not `kCGEventFlagMaskNonCoalesced` (`0x00000100`). No need to activate target app.
- AppKit side effect: `postToPid` sends event through target `sendEvent:` path; target window may become key while target app stays inactive. Clicker app remains frontmost; target `NSApplicationDidBecomeActiveNotification` should not fire.
- Cross-platform OSS architecture: Swift macOS MCP server + Go Windows/Linux runtimes. Windows uses PowerShell `System.Windows.Automation` + Win32 message fallback; Linux uses Python AT-SPI2 bridge. Same 9-tool protocol.
- Official Codex difference: OSS clone notes official service has caller signing/private IPC/session authorization; raw helper may fail with `Sender process is not authenticated`. Treat clone as behavior map, not proof of closed-source internals.
